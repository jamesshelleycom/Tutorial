## Development Conventions

### Code Patterns

1. **Interactive CLI Applications**
   - Menu-driven interfaces using infinite loops
   - Input validation with try-except blocks
   - Clear screen using ANSI escape codes (`\033c`)

2. **Data Persistence**
   - JSON-based data storage
   - File existence checking before reads
   - Graceful fallbacks to empty data structures

3. **Entry Points**
   - Main logic wrapped in `main_menu()` or similar functions
   - Direct execution pattern at file end

### File Naming

- Use snake_case for Python files
- Descriptive names indicating functionality
- Suffix `_recursive` for recursive algorithm variants

### Error Handling

```python
try:
    # Operation
except ValueError:
    print("Invalid input")
except FileNotFoundError:
    # Initialize empty data structure
except json.JSONDecodeError:
    # Handle corrupted data
```

## Code Style Guidelines

When modifying or adding code:

1. **Functions** - Use snake_case naming
2. **Variables** - Use descriptive snake_case names
3. **Strings** - Prefer double quotes for user-facing text
4. **Comments** - Add inline comments for complex logic
5. **Type Hints** - Consider adding for new code (not currently used)

## AI Assistant Guidelines

### When Adding New Features

1. Follow existing patterns for CLI applications
2. Use JSON for any data persistence needs
3. Include input validation and error handling
4. Keep code simple and educational

### When Modifying Existing Code

1. Maintain consistency with surrounding code style
2. Preserve interactive CLI behavior
3. Test manually after changes
4. Don't break existing JSON data format

### When Adding New Mini-Projects

1. Place in `Mini-projects/` directory
2. Use descriptive snake_case filename
3. Include a main function or entry point
4. Add input validation for user-facing features

### Common Tasks

- **Add a new algorithm:** Create in `Mini-projects/`, follow iterative/recursive naming pattern
- **Fix a bug:** Check for input validation and exception handling
- **Add persistence:** Use JSON with try-except for file operations

## Git Workflow

- **Main branch:** `main`
- **Commit style:** Simple descriptive messages ("Create filename.py", "Update filename.py")
- **No CI/CD:** Manual testing only

## Notes for AI Assistants

1. This is a learning repository - keep code simple and well-documented
2. All dependencies are from Python standard library
3. Directory names contain spaces (`Module Assessments/`) - quote paths in commands
4. Empty placeholder files exist in Module Assessments - they may need implementation
5. No `.gitignore` exists - avoid committing data files or virtual environments
